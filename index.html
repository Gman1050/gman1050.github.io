Ambrosia Archaeology VR

March 2017 - December 2018

HTC Vive, Oculus Rift / S

This was intially developed in Unity Engine, and then later ported to the Unreal Engine. It is a collaboration with the ATI Media Center and the Anthropology department at California State University San Bernardino. The goal for this project is to provide a simulation that provides the educational experience of the work an Archaeologist does. Transect Surveys are the main experience that the student will undergo, following encountering artifacts and examining the history and culture of each artifact. With VR, students can be transported to the virtual island known as Ambrosia and pick up artifacts with their hands and closely examine each one. The artifacts encountered in the simulation were created using a 3D modeling technique called photogrametry which was used to scan the artifacts that the Anthropology department has.

  • Learn More

Contributions:

  • Worked heavily on AI behavior of field study students that work with the VR user on the transect survey activities.
  • Implement the story telling of the transect survey activities using dialogue user interface tools.

Development Tools:

Dialogue Tool

Spring 2020

Unity Plugin

This was a personal project that I decided to make just in case I wanted to add dialogue to any game or simulation that I create in Unity. It is capable of producing linear dialogue trees, questions and answers that allow for multiple branch dialogue tree creations, looping dialogue trees, and audio clip support for each dialogue tree node.

This is also the first project where I learned how to use Unity's unit testing tool and provided fully-detailed documentation of this plugin. The current interface works with all canvas rendering modes: one for desktop and two for VR (SteamVR or UnityXR).

In the future, I plan to implement a visual editing interface that will replace the current scriptable object format that the dialogue trees are created in.

  • Github Page

VR S.O.C. Starting Prototype (Secure Operations Center)

CSUSB C.S. Senior Project class

January 2020 - March 2020

Windows, Oculus Quest

This was developed using Unreal Engine. This VR project was proposed for our Senior Project by a Computer Science student named Brett Evrist who also was in the Cyber Security program at CSUSB. The purpose of the VR S.O.C. is to simulate a real Secure Operations Center using the Unreal Engine 4. Our proposed contributions for the project as part of our overall class assignment was to deliver multiplayer, a S.O.C. room, keyboard interface for commands and text chat, monitor interfaces, and local voice chat. In conclusion, we were able to complete everything we proposed except for local voice chat. It is still in a imperfect state where voice gets recorded and played in the form of walkie-talkie, but not replicated over the network.

This project is created for and owned by the CSUSB CyberSecurity department so I won't be able to distribute it on my portfolio website. Repository is stored using SVN so you won't be able to see the source code.
